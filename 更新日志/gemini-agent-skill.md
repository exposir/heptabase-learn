# Gemini CLI 扩展代理技能 (Agent Skills)

> 本文件整理并维护自 [Gemini CLI Skills Documentation](https://geminicli.com/docs/cli/skills/)。

Agent Skills (代理技能系统) 赋予了 Gemini CLI 从专门化的技术角度、程序性作业流水线控制，乃至特定任务资源环境整合等方面更为宽阔的延展支持能力。立基于开放式的 Agent Skills 规范标准协议，所谓“一项技能” (a "skill")，实质上乃是一个经过自封装的完整文件目录，它打包封存着各类说明书及其它辅助资产数据，以可发现式的机制转化为平台可以直接驱使之技能操作集。

---

## 概述与理念 (Overview)

相区别于提供持久存留、涉及工作区范围广域常驻全景资料型 `GEMINI.md` 的存在形式，Skills 系统旨在表达一种需时随取的按需专家顾问机制 (on-demand expertise)。如此设计可让 Gemini 能够在绝佳之灵活性层面下存贮和维系一套极度庞大充沛的具备高度专门特长之操作合集——譬如自动化完成系统安全性审查操作流程 (security auditing)、复杂的跨级云主机部署流程 (cloud deployments) 又或者浩瀚庞大之前端底层代码基线迭代移植 (codebase migrations) —— 最核心的优势在于其并不会霸占填冲掉当前大型模型用以理解直层上下文的短窗口存储器容量 (cluttering the model's immediate context window)。

Gemini 的运行基核会基于接收到的各项操作指令及特定各个技能的描述宣告内容，自主评估判断是否应当启用介入相应的技能。一旦筛选甄别锁定下相关特定专门适用的技能，模型将会借由调用内置底层组件 `activate_skill` （激活一项专门技能）把所需的全部完整说明指令文与外部附加关联辅助运作的整套内容模块数据包全面抽调载入至当前处理执行逻辑空间以妥善完成最终的下发处理需要。

### 核心收益 (Key Benefits)

- **知识结晶分享 (Shared Expertise)**: 可将具有复杂交互逻辑属性的一套标准式运作流机制（这或许像是特属某个技术组长编订的繁复的拉取合并需求处理审批工作线：PR review process）归纳包纳进入一套独立资料夹体系内部，供所有的使用者随时取调使用。
- **工作流高维复查可用 (Repeatable Workflows)**: 透借赋予搭建一整套符合处理程序执行阶段性的指导架构方法论以确保能够针对高度复杂涉及多节点协同执行的深度需求实现连续及行为一致标准化的完成水准。
- **聚合配置资源打包管理 (Resource Bundling)**: 使各项系统自动脚本、格式填充范本或者示例演示素材库数据与执行主干引导说明说明被紧密附并绑定一起以防阻脱离环境导致的运行执行错误以让处理系统随时随处能够完配无虞。
- **递进阶解封释放架构 (Progressive Disclosure)**: 在一般运行时间层面，最初阶段仅仅只对特定各项专属技能的基底层原数据讯息 (仅包括其标定名称与说明大要描述) 实行基础加载动作而已。惟当这颗驱动基核经过评判并决意真正采取调令并触发实施那项特种专门对应的技能服务操作行为发生之时，相关更为详尽的深层指导与各种配属随行的资源群才会被正式予以全面曝露公开于运转进程的记忆读取之中，这么做的主要目的是以此来尽最大化去实现对于运作系统读取操作空间记忆资源用度层面的最大限度上解缩节约使用 (saving context tokens)。

## 专属技能溯源层级化搜索与分配定位法则 (Skill Discovery Tiers)

此项指令调用 CLI 的基本探索模块系统从三处特定的根本环境基数核心节点来发掘各项可用专属技能能力：

1.  **位于工作空间范畴内的特定专门应用技能库包资源体系 (Workspace Skills)**: 主要分布寄寓在此工程开发项目底端特定标称目录结构 `.gemini/skills/` 或是通过设置的虚拟桥接指引 `.agents/skills/` 中。此类型级的技术辅助配置大都会直接跟随着本尊基础主从源头项目本体进行提交推送从而归入统一全盘环境版本的管控之中去并作为可分享互通型态下发传播交付供给组内团队群体互联调用享用。
2.  **独归使用者自配层内技术特定应用系统组库储备设定资源包 (User Skills)**: 这些往往通常位于寄住在系统使用者根源路径 `~/.gemini/skills/` 以及相对设置相配接通的对应导引体系 `~/.agents/skills/` 。属于个人独享化使用的自备专门绝技能包系统资源，可以在全部跨不同项目的私人独控多维化项目矩阵集群里面去施展它以体现效用价值。
3.  **伴随系统外部附属引入挂架包体的扩建延伸补充形式技术配置合集库 (Extension Skills)**: 系统将一并探收与加载所有被一揽子合并打进到之前已经进行了本地运行布置与安装流程之延伸工具套件内含藏具备的技术专项组合资产能力集。

**执行选用顺次抢占准则 (Precedence)**: 倘若是面临同一标点称呼命名的不同专门技能力源于数个相异的环境空间被查寻获取时，则具有更加高一级先决判定权限定位级别来源层系的设置体系会发挥遮罩替代与霸道无理盖写的优先级效能并压制取代较低级顺位能力包；从强趋弱次序依次排列为：`Workspace` (本阵工作区) > `User` (用户底端体系) > `Extension` (随同系统衍生外壳包)。

同样针对于某个单一同等所属范畴内的相同空间中地带 (诸如都在用户自己底板或者同一项目区属阵地中时)，透过运用 `.agents/skills/` 这个路径指向的存取引流调用方式获取要凌上占据与超越过往原初之基源标准制式形式设立之存取通道名录结构 `.gemini/skills/` 的高判定优占顺序级别。这种更加偏走向趋合泛通用特质方向之配置指向型命名手段设定机制提供出了一条越加贴融直截合乎日常判断思考经验并更极佳地兼容且顺接各种差异类别人工系统工具控制的配置管理机制体系模式。

## 控制与驾驭你的技能装备库 (Managing Skills)

### 在互动形态命令行进程系统模式中应用操作法则 (In an Interactive Session)

使用者可以通过操作启用输入挂载于体系头端的带有导向标记杠指符之专属操作暗门命指令符项输入集合方法集方案系统套来直接查探调取配置相关特定特种专业体系库系统操作方案集合项目内容：

- `/skills list` _(缺省预设默认运作项目命令)_: 列陈并充分清晰展现给控制者目前已经被整体发现查明的全范围专门项目能力系统库配置名称状态项目及其分别各自独立对应之运作开启与关闭静默配置设置现制状态参数状况。
- `/skills link <path>`: 把一条连接接线导流系统纽带去建立连通于由外部指定给派向的本地存储放置体系内某个技术专用项体系资源目录系统使其得能够跨行过系统之底层环境桥梁架构并连点上通融入体系。
- `/skills disable <name>`: 使所点定对应的此名称指定体系功能特性陷闭进入禁用休克封禁阻断运作失效瘫痪处理不可取调用之冻结静候运行禁用锁状态制止调用应用处理。
- `/skills enable <name>`: 解锁释放把先前受困被执行关切隔断掉运行权限而遭处于静封瘫封闭禁状态设定所处理对付的一款针对性专属技能解冻重新唤激活。
- `/skills reload`: 去驱使指令要求命令平台框架针对已进行装载进入各个不相同探察追踪发掘层级地带的已知全种类可用技术支撑运作专用效能力集做一轮总体的通查重新提取加载上载焕新更新刷复整理。

> _要事关切附注 (Note)_: 此刻特别关涉如使用 `/skills disable` 和 `/skills enable` 此类型开关启闭操控类型设定之操作手段在一般通常操作中是径直将作用影响波及干涉指向属于控制此一设备的特定使用者自含属地权限领空操控管辖区域维度 `user scope` 系统框架之下。若有意图有需要将其操控施加效应转换指派去发生针对发生在这个整体项目工程运转基址范围内时，就需要特加采用 `--scope workspace` 作出更为精准的指向规定管界指派命令宣告方可生效达成该项专属管理设置动作要求规范诉求动作。

### 利用底层命令行系统程序操作框架运作驾驭技术支持库方案工具链指令 (From the Terminal)

通过脱离界面的 `gemini skills` 直通底层操作工具操作链也可以直对专门运作特化指令机制实施完全且高度集中的统括部署掌控调配权柄执行指挥管控管理行为操作配置处理系统管理动作操作实现功能应用实现支持：

```bash
# 执行查举并陈展展现被探获全部列陈上册之已知功能绝技
gemini skills list

# 创构设立象征导向链路（symlink）来导入远端地底资源进网
# 此方法探索追寻附含 （SKILL.md 及后续延伸 */SKILL.md 等规则设定包结构特征的资料夹系统库）并在属己之领域体系（ ~/.gemini/skills 或者 ~/.agents/skills 路径地址深处建立连接节点指路网导播导览系统
gemini skills link /path/to/my-skills-repo

# 利用指声明限定范围体系指令导通其链路挂上于该所在项目属组管盘辖地域内控运行统辖（对应产生挂搭影响效果落脚到 .gemini/skills 又或者到指向型代称映射路 .agents/skills 此地带区间范围
gemini skills link /path/to/my-skills-repo --scope workspace

# 直接运用网络安装或通过系统底层安装拉取解封入列新入专用项系统指令（其可以来源于远端 Git 网络架库系统源、主机磁盘上的物理储存区甚至被高度归总集拢压缩打定起来之后形成的 .skill 包档案资料形态
# （按照其原设内定规则缺省未声明时会以普通操控设备持有者本位层域权限区即 ~/.gemini/skills 抑或作代名引用的 ~/.agents/skills 系统作为挂驻宿附落点区）
gemini skills install https://github.com/user/repo.git
gemini skills install /path/to/local/skill
gemini skills install /path/to/local/my-expertise.skill

# 利用追加路径定向标识符 (--path) 这个挂配附加参数对置处于超大混成庞巨聚拢化集成组合体储存基址集仓库 (monorepo) 这内部某一幽微特异点目录区域位置处单向定向安装专门锁定需求抓取之子级附属配置专用套设定系统
gemini skills install https://github.com/my-org/my-skills.git --path skills/frontend-design

# 把执行拉取收容安装这整套行使工作实施的指定接受点划拨规范分配降落在本项目工程群集体管控归属地域之权限区盘范围之中进行（即为落子布置于该环境 .gemini/skills 或是同样作为替换标位符号作用机制体现存在着的 .agents/skills 的基准内区）
gemini skills install /path/to/skill --scope workspace

# 透过直接精准输入其具体之标定化点名移除摘除消减取缔解除一项专项效能特制技法技能其操作挂靠效法在（限定特指于工作所属区域框架内作用时
gemini skills uninstall my-expertise --scope workspace

# 给此相对应标点名称下的这一单类门特装特技作完全地全局解限松绑唤醒运行启挂复权运作（使得全维度通用运作可用可行化表现）
gemini skills enable my-expertise

# 令这一套配置专用能力系统项目进入断闭拒决拒止执行失效下封停止处理锁止禁用运作无能状态反应的处治指令操作应对处理。（可以通过配置设定指列选用 --scope 这个加挂跟随指示宣示词去确凿精确指定清楚到底是将其关闭之效果波及在工作群组的专属专区抑或者作为通常的系统拥有调拨指控权执行管控人之系统范畴下辖产生禁用作用效果（在不附加进行指特明确宣指时系统自动按默设的本环境之群构系统项目区域生效去运行发生改变判定））
gemini skills disable my-expertise --scope workspace
```

## 背后核心逻辑流理推衍体系大拆解剖析系统原理探看 (How it Works)

1.  **探采甄选辨识阶 (Discovery)**: 值处于某一连结活动操作时段区（session）方端开启建立那最初发轫端点的第一始时刻起， Gemini 这个系统化架构指令系统 CLI 将向全部拥有权限涉足的可容进入能够去寻找资源存在的对应搜索资源存放库范围区域阶层之中执行展开全部盘大范围大扫查大探巡操作进而会将经过检视审认其配置属于已正处启用合规受支持许可操作表现特质存在着状态正常良好的各项绝门技能配置中的名称字号代码项和相关精编撰明内容说明简介参数配置字元片段部分给集中提取汇总后一股脑强制灌注注入直系导入配置充实满填充填植种于给赋予这个当前操控中台环境大脑运转所需接收指引之总体纲标性系统根指令预引声明描述体系内 （system prompt）。
2.  **触击发动诱激阶段 (Activation)**: 将会发生一种自然状态运行机缘就在由于处于这之中的智能系统主体端 Gemini 通过自身智慧判定运作过程判断出了由外沿派递传发收发输送过来的动作行为标的工作处置目标任务与存列记载入底档之某一门绝密类特定技能系统参数里那一条功能说解参数特描述文字宣段特征之间产生了合拍配挂般吻合契对应点状态时候起触发运行这种效用机理之后，该智能平台程序则随即自己发动实施发起号召进行发号直接发下并启始呼传调调用其底层早已部署埋设齐配预存在那的一个名谓起称为专门针对特选专用技能解启开机促始机柄开关执行专用驱动程序的这个名叫 `activate_skill` 的内配组项功能操作工具进行调用运转行操作流转处理进行施术作业响应动作。
