# 1.2 异步编程专题

异步是 JavaScript 的灵魂。从 Callback Hell 到 Promise，再到 `async/await`，前端异步演进史本质上是**对"流程控制权"归还与语义化重塑的过程**。

---

### 1.2.1 Promise 系列 (Promise & Specification)

**现象**：异步操作可以在语法层面上像同步代码一样链性表达。
**本质**：通过状态机（Pending/Fulfilled/Rejected）管理回调函数的执行时机与结果传递。

| 题目                  | 难度       | 优先级 | 考察点                                       |
| :-------------------- | :--------- | :----- | :------------------------------------------- |
| **Promise 完整实现**  | ⭐⭐⭐⭐⭐ | 必会   | A+ 规范、状态迁移、then 链式调用、微任务队列 |
| **Promise 并发控制**  | ⭐⭐⭐⭐   | 必会   | 计数器控制、结果顺序保证、动态任务补位       |
| **Promise 重试/超时** | ⭐⭐⭐     | 必会   | 递归调用、指数退避策略、竞态处理             |
| **可取消的 Promise**  | ⭐⭐⭐     | 掌握   | AbortController 信号联动                     |

#### 1. Promise A+ 核心实现

实现 A+ 规范的关键在于 `then` 方法返回的是**全新的 Promise**，且 `resolvePromise` 过程必须处理 `thenable` 对象。

```javascript
// ─────────────────────────────────────────────────
//  MyPromise — 符合 A+ 规范的基础实现
// ─────────────────────────────────────────────────

const PENDING = "pending";
const FULFILLED = "fulfilled";
const REJECTED = "rejected";

class MyPromise {
  constructor(executor) {
    this.status = PENDING;
    this.value = undefined;
    this.reason = undefined;
    this.onResolvedCallbacks = [];
    this.onRejectedCallbacks = [];

    const resolve = (value) => {
      if (this.status === PENDING) {
        this.status = FULFILLED;
        this.value = value;
        this.onResolvedCallbacks.forEach((fn) => fn());
      }
    };

    const reject = (reason) => {
      if (this.status === PENDING) {
        this.status = REJECTED;
        this.reason = reason;
        this.onRejectedCallbacks.forEach((fn) => fn());
      }
    };

    try {
      executor(resolve, reject);
    } catch (e) {
      reject(e);
    }
  }

  then(onFulfilled, onRejected) {
    // 穿透处理
    onFulfilled = typeof onFulfilled === "function" ? onFulfilled : (v) => v;
    onRejected =
      typeof onRejected === "function"
        ? onRejected
        : (e) => {
            throw e;
          };

    const promise2 = new MyPromise((resolve, reject) => {
      if (this.status === FULFILLED) {
        queueMicrotask(() => {
          try {
            const x = onFulfilled(this.value);
            resolvePromise(promise2, x, resolve, reject);
          } catch (e) {
            reject(e);
          }
        });
      }

      if (this.status === REJECTED) {
        queueMicrotask(() => {
          try {
            const x = onRejected(this.reason);
            resolvePromise(promise2, x, resolve, reject);
          } catch (e) {
            reject(e);
          }
        });
      }

      if (this.status === PENDING) {
        this.onResolvedCallbacks.push(() => {
          queueMicrotask(() => {
            try {
              const x = onFulfilled(this.value);
              resolvePromise(promise2, x, resolve, reject);
            } catch (e) {
              reject(e);
            }
          });
        });
        this.onRejectedCallbacks.push(() => {
          queueMicrotask(() => {
            try {
              const x = onRejected(this.reason);
              resolvePromise(promise2, x, resolve, reject);
            } catch (e) {
              reject(e);
            }
          });
        });
      }
    });

    return promise2;
  }
}

function resolvePromise(promise2, x, resolve, reject) {
  // 1. 循环引用判定
  if (promise2 === x) {
    return reject(new TypeError("Chaining cycle detected for promise"));
  }

  // 2. 处理 thenable 对象
  if ((x !== null && typeof x === "object") || typeof x === "function") {
    let called = false;
    try {
      let then = x.then;
      if (typeof then === "function") {
        then.call(
          x,
          (y) => {
            if (called) return;
            called = true;
            resolvePromise(promise2, y, resolve, reject);
          },
          (r) => {
            if (called) return;
            called = true;
            reject(r);
          },
        );
      } else {
        resolve(x);
      }
    } catch (e) {
      if (called) return;
      called = true;
      reject(e);
    }
  } else {
    resolve(x);
  }
}

// ── 验证 ──
new MyPromise((resolve) => {
  setTimeout(() => resolve("Success"), 100);
})
  .then((res) => {
    console.log(res); // "Success"
    return "Next";
  })
  .then((res) => console.log(res)); // "Next"
```

> **本质洞察**：`Promise` 的 `then` 链条不是把 callback 存起来那么简单，它是在构建一颗**状态分叉树**。每一个 `.then` 都是对一个新世界的承诺。

### 1.2.2 异步控制流 (Async Flow Control)

**现象**：需要并发抓取 100 个接口，但浏览器限制只能同时发 6 个请求。
**本质**：利用**生产者-消费者模型**（Queue）或计数器实现对并发槽位的精准占用与释放。

#### 并发任务调度器 Scheduler

```javascript
// ─────────────────────────────────────────────────
//  Scheduler — 带并发限制的任务调度器
// ─────────────────────────────────────────────────

class Scheduler {
  constructor(limit) {
    this.limit = limit;
    this.runningCount = 0;
    this.queue = [];
  }

  add(taskCreator) {
    return new Promise((resolve) => {
      // 包装任务，执行完后通知 Resolve
      const task = async () => {
        const result = await taskCreator();
        resolve(result);
        this.next(); // 释放槽位，执行下一个
      };

      this.queue.push(task);
      this.run();
    });
  }

  run() {
    while (this.runningCount < this.limit && this.queue.length > 0) {
      const task = this.queue.shift();
      this.runningCount++;
      task();
    }
  }

  next() {
    this.runningCount--;
    this.run();
  }
}

// ── 验证 ──
const timeout = (time) => new Promise((resolve) => setTimeout(resolve, time));
const scheduler = new Scheduler(2);

const addTask = (time, order) => {
  scheduler.add(() => timeout(time)).then(() => console.log(order));
};

addTask(1000, "1");
addTask(500, "2");
addTask(300, "3");
addTask(400, "4");
// 输出顺序: 2, 3, 1, 4 (2并1，2完3进，3完4进，1慢最后)
```

> **本质洞察**：并发控制的精髓不在于"如何运行"，而在于"如何等待"。调度器通过 `Promise` 无限期阻塞外部调用，直到内部队列腾出空间。

### 1.2.3 定时器专题 (Timers & Optimization)

| 题目              | 难度   | 优先级 | 考察点                             |
| :---------------- | :----- | :----- | :--------------------------------- |
| **防抖 debounce** | ⭐⭐   | 必会   | 闭包、定时器重置、立即执行支持     |
| **节流 throttle** | ⭐⭐   | 必会   | 时间戳与定时器的混合方案、首尾调用 |
| **RAF 节流**      | ⭐⭐⭐ | 掌握   | `requestAnimationFrame` 适配高刷屏 |

#### 完整版防抖 (Debounce)

```javascript
function debounce(fn, wait, immediate = false) {
  let timer = null;

  return function (...args) {
    const context = this;
    if (timer) clearTimeout(timer);

    if (immediate && !timer) {
      fn.apply(context, args);
    }

    timer = setTimeout(() => {
      if (!immediate) fn.apply(context, args);
      timer = null;
    }, wait);
  };
}
```

## 质量指标

- **微任务一致性**：`MyPromise` 必须使用 `queueMicrotask` 或 `MutationObserver` 模拟异步。
- **内存安全**：`Scheduler` 任务完成后必须能被垃圾回收。
- **稳定性**：`debounce` 必须处理 `this` 指向及参数透传。
