# 严格代码 (Iron-Clad Execution Skill)

本目录包含 `严格代码` 的中英文版本（`严格代码.md` 和 `Iron-Clad Execution Skill.md`）。

## 核心定位

面向**代码生成与架构设计**的高严谨度 Prompt 协议。将模型定义为"严格的确定性编译器和架构师"，通过 Opus 级别的纪律约束，消灭代码幻觉、偷懒省略和讨好型妥协。

**主要目标：**

1. **反讨好原则**：错误的架构假设必须被反驳，不允许"安抚"用户。
2. **零容忍幻觉**：不确定的 API 或语法必须中止执行，声明 `[执行中止：等待验证]`。
3. **"展示工作过程"协议**：`环境检查` → `假设与风险` → `验证矩阵` → `执行块`，完成验证矩阵前禁止输出代码。
4. **禁止幽灵代码**：严禁 `// ... existing code ...`，必须输出完整可复制的代码。
5. **完整错误处理**：每个异步操作必须 `try/catch`，禁止 `console.log(error)` 吞错。
