# 铁壁执行技能指南 (Iron-Clad Execution Skill)

本目录包含 `铁壁执行技能指南` 的中英文版本（`铁壁执行技能指南.md` 和 `Iron-Clad Execution Skill.md`）。

## 核心定位

这是一个**面向代码生成与架构设计**的高严谨度 Prompt 协议。它将模型重新定义为"严格的确定性编译器和架构师"，通过 Opus 级别的纪律约束，消灭代码幻觉、偷懒省略和讨好型妥协。

**主要目标：**

1. **反讨好原则**：错误的架构假设必须被反驳，不允许"安抚"用户。
2. **零容忍幻觉**：不确定的 API、语法或端点必须中止执行并声明 `[执行中止：等待验证]`，绝不猜测。
3. **"展示工作过程"协议**：强制序列化推理路径（`环境检查` → `假设与风险` → `验证矩阵` → `执行块`），在完成验证矩阵前禁止输出代码。
4. **禁止幽灵代码**：严禁 `// ... existing code ...` 等省略写法，必须输出完整可复制的代码。
5. **完整错误处理**：每个异步操作必须包含 `try/catch`，禁止 `console.log(error)` 吞噬错误。

## 核心机制设计哲学

1. **"展示工作过程"反捷径机制**
   当约束条件超过 5 项时，Gemini 模型倾向走捷径。本协议通过强制的 4 步序列化推理路径（先验证再执行），物理性地阻断模型跳过约束条件的通路。

2. **Opus 级别代码严谨性**
   借鉴 Claude Opus 的代码生成纪律，要求输出完整函数体、强制错误处理、禁止凭空导入未验证的库。

3. **高复杂度推理释放**
   在严谨性基础上，鼓励充分利用 1M token 窗口进行深层上下文整合、追溯数据流、采用高级 React 模式和 O(N) 算法效率。
