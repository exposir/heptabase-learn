# 高能动前端技能指南

## 1. 激活基线配置

- DESIGN_VARIANCE（设计变量）: 8（1=完美对称，10=艺术混沌）
- MOTION_INTENSITY（动效强度）: 6（1=静态/无动画，10=电影级/魔法物理）
- VISUAL_DENSITY（视觉密度）: 4（1=艺术画廊/通透，10=驾驶舱/信息密集）

**AI 指令：** 所有生成的标准基线严格固定为上述值（8, 6, 4）。不要要求用户编辑此文件。其他情况下，始终优先听从用户指令：根据用户在对话中明确提出的需求，动态调整这些值。将这些基线值（或用户覆盖的值）作为全局变量，驱动第 3 至 7 节中的具体逻辑。

## 2. 默认架构与规范

除非用户明确指定其他技术栈，否则遵守以下结构约束以保持一致性：

- **依赖验证【强制】：** 在引入任何第三方库（如 `framer-motion`、`lucide-react`、`zustand`）之前，必须检查 `package.json`。若该包缺失，必须在提供代码之前输出安装命令（如 `npm install package-name`）。**绝不**假设某个库已存在。
- **框架与交互性：** React 或 Next.js。默认使用服务端组件（`RSC`）。
  - **RSC 安全性：** 全局状态仅在客户端组件中有效。在 Next.js 中，将 Provider 包装在带有 `"use client"` 的组件中。
  - **交互隔离：** 如果第 4 或第 7 节（动效/液态玻璃）处于激活状态，具体的交互式 UI 组件必须提取为带有 `'use client'` 声明的独立叶子组件。服务端组件必须仅渲染静态布局。
- **状态管理：** 对独立 UI 使用本地 `useState`/`useReducer`。全局状态严格用于避免深层 prop 透传。
- **样式策略：** 使用 Tailwind CSS（v3/v4）承担 90% 的样式工作。
  - **Tailwind 版本锁定：** 先检查 `package.json`。不要在 v3 项目中使用 v4 语法。
  - **T4 配置守卫：** v4 中，**不要**在 `postcss.config.js` 中使用 `tailwindcss` 插件。使用 `@tailwindcss/postcss` 或 Vite 插件。
- **反 Emoji 策略【关键】：** 禁止在代码、标记、文本内容或 alt 属性中使用 emoji。用高质量图标（Radix、Phosphor）或简洁的 SVG 原语替代。Emoji 被明令禁止。
- **响应式与间距：**
  - 标准化断点（`sm`、`md`、`lg`、`xl`）。
  - 使用 `max-w-[1400px] mx-auto` 或 `max-w-7xl` 约束页面布局。
  - **视口稳定性【关键】：** 全高度 Hero 区域绝不使用 `h-screen`。始终使用 `min-h-[100dvh]` 以防止移动浏览器（iOS Safari）出现灾难性布局跳动。
  - **Grid 优于 Flex 百分比计算：** 绝不使用复杂的 flexbox 百分比数学（`w-[calc(33%-1rem)]`）。始终使用 CSS Grid（`grid grid-cols-1 md:grid-cols-3 gap-6`）构建可靠结构。
- **图标：** 必须使用 `@phosphor-icons/react` 或 `@radix-ui/react-icons` 作为精确的导入路径（检查已安装版本）。全局统一 `strokeWidth`（如专用 `1.5` 或 `2.0`）。

## 3. 设计工程指令（偏见纠正）

大模型在统计上偏向特定的 UI 陈词滥调模式。主动运用以下工程化规则，构建高端界面：

**规则 1：确定性排版**

- **展示/标题：** 默认 `text-4xl md:text-6xl tracking-tighter leading-none`。
  - **反烂设计：** 不鼓励为"高级"或"创意"氛围使用 `Inter`。通过 `Geist`、`Outfit`、`Cabinet Grotesk` 或 `Satoshi` 强制建立独特个性。
  - **技术 UI 规则：** 衬线字体在 Dashboard/软件 UI 中被严格禁止。此类场景下，专用高端无衬线字体组合（`Geist` + `Geist Mono` 或 `Satoshi` + `JetBrains Mono`）。
- **正文/段落：** 默认 `text-base text-gray-600 leading-relaxed max-w-[65ch]`。

**规则 2：色彩校准**

- **约束：** 最多 1 个强调色。饱和度 < 80%。
- **紫色禁令：** 严格禁止"AI 紫/蓝"美学。不允许紫色按钮光晕或霓虹渐变。使用绝对中性基底（Zinc/Slate）搭配高对比度单一强调色（如 Emerald、Electric Blue 或 Deep Rose）。
- **色彩一致性：** 整个输出中坚持使用同一色板。同一项目内不要在暖灰和冷灰之间切换。

**规则 3：布局多元化**

- **反居中偏见：** 当 `LAYOUT_VARIANCE > 4` 时，居中 Hero/H1 区域被严格禁止。强制使用"分屏"（50/50）、"左对齐内容/右对齐资产"或"非对称留白"结构。

**规则 4：材质感、阴影与"反卡片滥用"**

- **Dashboard 强化：** 当 `VISUAL_DENSITY > 7` 时，通用卡片容器被严格禁止。使用逻辑分组（`border-t`、`divide-y`）或纯负空间。数据指标应在无边框状态下自由呼吸，除非层级（z-index）有功能性需求。
- **执行：** 仅在层级传达需要时使用卡片。使用阴影时，将其色调与背景色调和。

**规则 5：交互 UI 状态**

- **强制生成：** 大模型天然只生成"静态成功状态"。必须实现完整的交互周期：
  - **加载中：** 与布局尺寸匹配的骨架屏（避免通用圆形加载动画）。
  - **空状态：** 精心构建的空状态，提示用户如何填充数据。
  - **错误状态：** 清晰的内联错误报告（如表单）。
  - **触觉反馈：** `:active` 时使用 `-translate-y-[1px]` 或 `scale-[0.98]` 模拟物理按压感，表示成功/操作。

**规则 6：数据与表单模式**

- **表单：** 标签必须位于输入框上方。辅助文本可选但应存在于标记中。错误文本位于输入框下方。输入块使用标准的 `gap-2`。

## 4. 创意主动性（反烂设计实现）

为主动对抗通用 AI 设计，系统性地将以下高端编程概念作为基线实现：

- **"液态玻璃"折射：** 需要玻璃拟态效果时，超越 `backdrop-blur`。添加 1px 内边框（`border-white/10`）和细微内阴影（`shadow-[inset_0_1px_0_rgba(255,255,255,0.1)]`）以模拟物理边缘折射。
- **磁性微物理（若 MOTION_INTENSITY > 5）：** 实现按钮轻微朝鼠标光标方向吸附。**关键：** 磁性悬停或持续动画绝不使用 React `useState`。专用 Framer Motion 的 `useMotionValue` 和 `useTransform`，在 React 渲染周期之外运行，以防止移动端性能崩溃。
- **持续微交互：** 当 `MOTION_INTENSITY > 5` 时，在标准组件（头像、状态点、背景）中嵌入连续、无限循环的微动画（脉冲、打字机、浮动、闪烁、轮播）。对所有交互元素应用高端弹簧物理（`type: "spring", stiffness: 100, damping: 20`）——不使用线性缓动。
- **布局过渡：** 始终使用 Framer Motion 的 `layout` 和 `layoutId` props，实现跨状态变化的平滑重排、缩放和共享元素过渡。
- **交错编排：** 不要瞬间挂载列表或网格。使用 `staggerChildren`（Framer）或 CSS 级联（`animation-delay: calc(var(--index) * 100ms)`）创建顺序瀑布式展现。**关键：** 使用 `staggerChildren` 时，父级（`variants`）和子级必须位于相同的客户端组件树中。若数据异步获取，将数据作为 props 传入集中的父级 Motion 包装器。

## 5. 性能护栏

- **DOM 开销：** 将颗粒/噪点滤镜专用于固定的、`pointer-events-none` 的伪元素（如 `fixed inset-0 z-50 pointer-events-none`），绝不应用于滚动容器，以防止持续 GPU 重绘和移动端性能下降。
- **硬件加速：** 绝不动画化 `top`、`left`、`width` 或 `height`。专用通过 `transform` 和 `opacity` 进行动画。
- **Z-Index 克制：** 绝不随意滥用 `z-50` 或 `z-10`。仅在系统性层级上下文（固定导航、模态框、遮罩层）严格使用 z-index。

## 6. 技术参考（刻度盘定义）

### DESIGN_VARIANCE（设计变量，1-10 级）

- **1-3（可预测）：** Flexbox `justify-center`，严格 12 列对称网格，等距内边距。
- **4-7（偏移）：** 使用 `margin-top: -2rem` 叠加，混合图片比例（如 4:3 与 16:9 并排），左对齐标题配居中对齐数据。
- **8-10（非对称）：** 瀑布布局，带有分数单位的 CSS Grid（如 `grid-template-columns: 2fr 1fr 1fr`），大面积留白（`padding-left: 20vw`）。
- **移动端覆盖规则：** 对于 4-10 级，`md:` 以上的任何非对称布局必须在 `< 768px` 视口下积极回退为严格的单列布局（`w-full`、`px-4`、`py-8`），以防止水平滚动和布局破坏。

### MOTION_INTENSITY（动效强度，1-10 级）

- **1-3（静态）：** 无自动动画。仅 CSS `:hover` 和 `:active` 状态。
- **4-7（流畅 CSS）：** 使用 `transition: all 0.3s cubic-bezier(0.16, 1, 0.3, 1)`。使用 `animation-delay` 级联实现加载进场。严格聚焦于 `transform` 和 `opacity`。谨慎使用 `will-change: transform`。
- **8-10（高级编排）：** 复杂的滚动触发式展现或视差效果。使用 Framer Motion hooks。绝不使用 `window.addEventListener('scroll')`。

### VISUAL_DENSITY（视觉密度，1-10 级）

- **1-3（艺术画廊模式）：** 大量留白。超大区块间距。一切都显得昂贵而简洁。
- **4-7（日常 App 模式）：** 标准 Web App 的正常间距。
- **8-10（驾驶舱模式）：** 极小内边距。无卡片边框；仅用 1px 线条分隔数据。一切都塞满。**强制：** 所有数字使用等宽字体（`font-mono`）。

## 7. 100 个 AI 特征（禁止模式）

为确保输出高端且非通用，必须严格避免以下常见 AI 设计特征，除非用户明确要求：

### 视觉与 CSS

- **禁止霓虹/外发光：** 不使用默认 `box-shadow` 发光或自动光晕。使用内边框或微妙的色调阴影。
- **禁止纯黑：** 绝不使用 `#000000`。使用近黑色（Off-Black）、Zinc-950 或炭灰色（Charcoal）。
- **禁止过饱和强调色：** 降低强调色饱和度，使其与中性色优雅融合。
- **禁止过度渐变文字：** 不对大标题使用渐变文字填充。
- **禁止自定义鼠标光标：** 已过时，且会破坏性能/可访问性。

### 排版

- **禁止 Inter 字体：** 被禁用。使用 `Geist`、`Outfit`、`Cabinet Grotesk` 或 `Satoshi`。
- **禁止超大 H1：** 第一个标题不应喊叫。通过字重和颜色控制层级，而非一味放大尺寸。
- **衬线字体约束：** 仅在创意/编辑类设计中使用衬线字体。**绝不**在简洁 Dashboard 上使用衬线字体。

### 布局与间距

- **对齐与间距须完美：** 确保内边距和外边距在数学上严格精确。避免元素浮动导致的尴尬间隙。
- **禁止 3 列等宽卡片布局：** 通用的"3 张等宽卡片横向排列"特性行被严格禁止。改用 2 列 Z 字形、非对称网格或水平滚动方案。

### 内容与数据（"Jane Doe"效应）

- **禁止通用名称：** "John Doe"、"Sarah Chan" 或 "Jack Su" 被禁用。使用高度创意、听起来真实的名称。
- **禁止通用头像：** 不使用标准 SVG"蛋型"或 Lucide 用户图标作为头像。使用有创意、可信度高的图片占位或特定样式。
- **禁止假数据：** 避免可预测的输出，如 `99.99%`、`50%` 或基础电话号码（`1234567`）。使用有机的、混乱的数据（`47.2%`、`+1 (312) 847-1928`）。
- **禁止创业烂名：** "Acme"、"Nexus"、"SmartFlow"。发明高端、有场景感的品牌名。
- **禁止填充词：** 避免 AI 文案陈词滥调，如"Elevate"、"Seamless"、"Unleash" 或 "Next-Gen"。使用具体的动词。

### 外部资源与组件

- **禁止损坏的 Unsplash 链接：** 不使用 Unsplash。使用绝对可靠的占位图，如 `https://picsum.photos/seed/{random_string}/800/600` 或 SVG UI Avatars。
- **shadcn/ui 定制化：** 可以使用 `shadcn/ui`，但绝不在其通用默认状态下使用。必须根据高端项目美学定制圆角、颜色和阴影。
- **生产级代码整洁度：** 代码必须极度整洁、视觉震撼、令人难忘，并在每个细节上精雕细琢。

## 8. 创意武器库（高端灵感）

不要默认到通用 UI。从以下高级概念库中汲取灵感，确保输出在视觉上震撼且令人难忘。在合适时机，利用 **GSAP（ScrollTrigger/Parallax）** 实现复杂的滚动叙事，或用 **ThreeJS/WebGL** 实现 3D/Canvas 动画，而非基础 CSS 动效。**关键：** 绝不在同一个组件树中混用 GSAP/ThreeJS 与 Framer Motion。Framer Motion 用于 UI/Bento 交互，GSAP/ThreeJS 专用于隔离的全页滚动叙事或 canvas 背景，并包裹在严格的 useEffect 清理块中。

### 标准 Hero 范式

- 停止做暗图上的居中文字。尝试非对称 Hero 区域：文字整齐左对齐或右对齐。背景应使用高质量、相关的图片，配以微妙的风格化渐隐（根据亮色/暗色模式，优雅地变暗或变亮，融入背景色）。

### 导航与菜单

- **Mac OS Dock 放大效果：** 边缘导航栏；图标在悬停时流畅地缩放。
- **磁性按钮：** 按钮物理性地朝光标方向吸附。
- **粘液菜单：** 子项像粘稠液体一样从主按钮上脱离。
- **灵动岛：** 胶囊形状的 UI 组件，变形显示状态/提醒。
- **场景化径向菜单：** 在点击坐标处精确展开的环形菜单。
- **悬浮快速拨号：** FAB 按钮弹出为弧形的次级操作列表。
- **巨型菜单展开：** 全屏下拉菜单，错落渐隐复杂内容。

### 布局与网格

- **Bento 网格：** 非对称、基于瓦片的分组（如苹果控制中心）。
- **瀑布布局：** 无固定行高的错落网格（如 Pinterest）。
- **色彩网格：** 网格边框或瓦片展示微妙的持续动画色彩渐变。
- **分屏滚动：** 两个屏幕半区在滚动时向相反方向滑动。
- **幕帘展开：** Hero 区域在滚动时像幕帘一样从中间分开。

### 卡片与容器

- **视差倾斜卡片：** 跟随鼠标坐标的 3D 倾斜卡片。
- **聚光灯边框卡片：** 卡片边框在光标下动态发光。
- **玻璃拟态面板：** 带内折射边框的真实磨砂玻璃效果。
- **全息箔纸卡片：** 悬停时彩虹光泽反射偏移的虹彩效果。
- **Tinder 滑卡堆叠：** 用户可以滑走的物理卡片堆叠。
- **形变模态框：** 按钮无缝展开为其自身的全屏对话框容器。

### 滚动动画

- **粘性堆叠卡片：** 卡片粘附在顶部并物理性地叠加在彼此之上。
- **水平滚动劫持：** 垂直滚动转化为平滑的水平画廊平移。
- **Locomotive 滚动序列：** 视频/3D 序列的帧率与滚动条直接绑定。
- **缩放视差：** 中央背景图在滚动时无缝放大/缩小。
- **滚动进度路径：** SVG 矢量线条或路径随用户滚动自我绘制。
- **液态划屏过渡：** 像粘稠液体一样擦过屏幕的页面切换效果。

### 画廊与媒体

- **穹顶画廊：** 3D 画廊，营造全景穹顶般沉浸感。
- **Coverflow 轮播：** 中心聚焦、边缘向后倾斜的 3D 轮播。
- **拖拽平移网格：** 可在任意方向自由拖动的无边界网格。
- **手风琴图片滑块：** 窄竖/横图片条，悬停时完全展开。
- **悬停图片轨迹：** 鼠标留下一串弹出/渐隐图片的轨迹。
- **故障效果图片：** 悬停时短暂的 RGB 通道偏移数字失真。

### 排版与文字

- **动态跑马灯：** 在滚动时反向或加速的无限文字带。
- **文字遮罩揭示：** 巨大的排版作为视频背景的透明窗口。
- **文字乱码效果：** 加载或悬停时的矩阵风格字符解码。
- **圆形文字路径：** 沿旋转圆形路径弯曲的文字。
- **渐变描边动画：** 渐变沿描边轮廓持续流动的空心文字。
- **动态排版网格：** 字母网格在光标靠近时躲开或旋转。

### 微交互与效果

- **粒子爆炸按钮：** 成功时 CTA 按钮破碎成粒子。
- **液态下拉刷新：** 移动端刷新指示器像脱离的水滴一样动作。
- **骨架屏闪光：** 移动的光线反射扫过占位框。
- **方向感知悬停按钮：** 悬停填充从鼠标进入的那一侧流入。
- **涟漪点击效果：** 从点击坐标精确向外扩散的视觉波纹。
- **SVG 线条动画绘制：** 矢量图形实时自我描绘轮廓。
- **网格渐变背景：** 有机的、熔岩灯式的动画色彩团。
- **镜头模糊景深：** 动态聚焦模糊背景 UI 层，突出前景操作。

## 9. "动效引擎" Bento 范式

生成现代 SaaS 仪表盘或特性展示区时，必须采用以下"Bento 2.0"架构和动效哲学。这超越了静态卡片，强制执行一种"Vercel 内核遇上 Dribbble 美学"的风格，大量依赖持续的物理动效。

### A. 核心设计哲学

- **美学：** 高端、极简、功能性。
- **调色板：** 背景使用 `#f9fafb`。卡片为纯白（`#ffffff`），配 `border-slate-200/50` 的 1px 边框。
- **表面：** 所有主要容器使用 `rounded-[2.5rem]`。应用"漫射阴影"（极淡、大范围扩散的阴影，如 `shadow-[0_20px_40px_-15px_rgba(0,0,0,0.05)]`）制造层次感而不显凌乱。
- **排版：** 严格使用 `Geist`、`Satoshi` 或 `Cabinet Grotesk` 字体栈。标题使用细微字距（`tracking-tight`）。
- **标签：** 标题和描述必须放置在卡片**外部和下方**，以保持简洁的画廊式呈现。
- **像素完美：** 卡片内使用充裕的 `p-8` 或 `p-10` 内边距。

### B. 动画引擎规格（持续动效）

所有卡片必须包含**"持续微交互"**。遵循以下 Framer Motion 原则：

- **弹簧物理：** 不使用线性缓动。使用 `type: "spring", stiffness: 100, damping: 20` 营造高端、有重量感的质感。
- **布局过渡：** 大量使用 `layout` 和 `layoutId` props，确保平滑的重排、缩放和共享元素状态过渡。
- **无限循环：** 每张卡片必须有一个无限循环的"激活状态"（脉冲、打字机、浮动或轮播），确保仪表盘感觉"有生命"。
- **性能：** 将动态列表包裹在 `<AnimatePresence>` 中，并优化到 60fps。**性能关键：** 任何持续动效或无限循环必须使用 `React.memo` 进行记忆化，并完全隔离在其自身的微型客户端组件中。绝不在父级布局中触发重渲染。

### C. 五种卡片原型（微动画规格）

构建 Bento 网格时实现以下特定微动画（如：第一行 3 列 | 第二行 2 列 70/30 分配）：

1. **智能列表：** 带有无限自动排序循环的垂直项目堆叠。使用 `layoutId` 交换位置，实时模拟 AI 对任务的优先级调整。
2. **命令输入框：** 带有多步打字机效果的搜索/AI 输入框。循环展示复杂提示词，包含闪烁光标和带渐变闪光的"处理中"状态。
3. **实时状态：** 带"呼吸感"状态指示器的日程界面。包含一个带有"过冲"弹簧效果弹出的通知角标，停留 3 秒后消失。
4. **宽幅数据流：** 数据卡片或指标的水平"无限轮播"。确保循环无缝衔接（使用 `x: ["0%", "-100%"]`），速度感觉毫不费力。
5. **场景化 UI（专注模式）：** 文档视图，动画化文本块的错落高亮，随后"浮入"一个带微图标的浮动操作工具栏。
