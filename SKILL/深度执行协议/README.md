# 深度执行协议 (Deep Execution Protocol)

本目录包含 `深度执行协议` 的中英文版本（`深度执行协议.md` 和 `Deep Execution Protocol.md`）。

## 核心定位

这是一个**纯行为约束型**的 Prompt 协议。它不赋予模型特定的角色或人格，而是通过强硬的指令规则，逼迫模型突破常见的“偷懒”、“谄媚”和“幻觉”倾向。

**主要目标：**

1. **强制事实锚定**：通过联网搜索作为基准，消灭知识盲区。
2. **消灭信息熵**：结论先行，不说废话。
3. **突破深度天花板**：强迫模型将 `90%+` 的算力（Token）投入到多维度、颗粒度极细的交叉展开中。
4. **防降维承诺机制**：利用 LLM 的自回归特性，强制在输出初期立下“规模字据”，利用一致性偏好防止后续输出缩水。
5. **对抗自我阉割**：摧毁模型为避免截断而擅自对内容进行压缩的底层倾向。

## 核心机制设计哲学

本协议之所以有效，是因为它精准狙击了 LLM 底层的几个行为偏好：

1. **“前置预估”防缩水**
   LLM 具有强烈的“上下文一致性偏好”（自回归生成特性）。如果在回答的最后才判断要不要分段，模型往往会提前开始总结。本协议**强迫模型在生成正文前，先宣告预估输出规模**（如 `【极大】`）。一旦这几个字生成，模型为了维持逻辑自洽，会被迫在后续生成中保持相应的长篇幅。这是对抗“偷懒”最有效的 prompt 工程技巧。

2. **“信息密度测试”防兑水**
   要求模型自己衡量：“这段话删掉后是否损失具体信息？”这把“不要写废话”这种软性诉求，变成了一个可操作的判别规则。

## 适用场景

适用于任何需要**极高信息密度**、**深度剖析**且**答案开放**的复杂任务：

- 行业深度研究报告
- 复杂技术选型与架构溯源
- 历史/政治/经济事件的多维度推演
- 源码级别的底层运作原理解析

## 多模型适配性与已知副作用

本协议与模型解耦，但在目前的一线大模型（Gemini 1.5/3.1 Pro, Claude 3.5 Sonnet / 3 Opus）上表现会有微妙差异。

### 对 Gemini 的表现

Gemini 系列（尤其是 Pro/Ultra 模型）**最契合此协议**。

- **匹配点**：Gemini 原生支持强大的 Google 搜索整合，能完美执行协议第一步的交叉验证。Gemini 的原生上下文窗口极大，分段输出机制能最大化压榨其长文本能力。
- **副作用**：如果搜索结果冲突，Gemini 偶尔会因为努力调和矛盾而导致输出结构变得臃肿。

### 对 Claude 的表现与“副作用对冲”

Claude 家族（尤其是 Sonnet 3.5）本身已经具备极高的严谨性和较少的幻觉，叠加此协议会产生以下**副作用**：

1. **“过度防御症” (Over-defensiveness)**
   - **症状**：因为协议中严格的《零幻觉协议》，Claude 可能会极度保守。明明是高置信度的常识或合理的逻辑推断，它也可能疯狂打上 `[推测]` 或 `[未验证]` 的标签，导致最终输出像一份满是免责声明的法务合同。
   - **缓解策略**：如果你发现 Claude 出现此症状，可以在使用此 Skill 时附加一句临时指令：“对于具备行业共识或高置信度的常识，无需过度标记不确定性。”

2. **信息密度稀释现象**
   - **症状**：Claude 默认的输出倾向是“精炼、直切主题”。协议强迫它把 90% 的 Tokens 花在展开上（逼迫它写长）。为了凑够篇幅并满足多维度要求，Claude 可能会用华丽的句式去稀释原本高密度的信息。
   - **缓解策略**：协议中的《信息密度检查》（“删掉后不损失具体信息的段落即为冗余”）专门用于对抗此倾向。这能逼迫它填充事实而非修辞。

3. **联网能力的缺失依赖**
   - **症状**：如果是基础的 Claude API 或网页版（未开启外接搜索时），它无法执行第一步的“强制联网校验”。
   - **缓解策略**：协议中已设定了降级路径——它会先声明 `[联网校验：未获取到可靠信息]`，然后基于预训练权重继续执行深度展开。这不会导致任务失败。
