# 深入执行 (Deep Execution Protocol)

本目录包含 `深入执行` 的中英文版本（`深入执行.md` 和 `Deep Execution Protocol.md`）。

## 核心定位

**纯行为约束型** Prompt 协议。不赋予模型特定角色或人格，而是通过强硬的指令规则，逼迫模型突破"偷懒"、"谄媚"和"幻觉"倾向。

**主要目标：**

1. **强制事实锚定**：通过联网搜索作为基准，消灭知识盲区。
2. **消灭信息熵**：结论先行，不说废话。
3. **突破深度天花板**：强迫模型将 `90%+` 的 Token 投入到多维度、颗粒度极细的交叉展开中。
4. **防降维承诺机制**：利用 LLM 的自回归特性，强制在输出初期立下"规模字据"，防止后续缩水。
5. **对抗自我阉割**：摧毁模型为避免截断而擅自压缩内容的底层倾向。

## 核心机制设计哲学

1. **"前置预估"防缩水**
   LLM 具有强烈的"上下文一致性偏好"。本协议强迫模型在生成正文前先宣告预估输出规模（如 `【极大】`）。一旦生成，模型为维持逻辑自洽，会被迫保持相应的长篇幅。

2. **"信息密度测试"防兑水**
   要求模型自己衡量："这段话删掉后是否损失具体信息？"把反废话诉求变成了可操作的判别规则。

## 适用场景

- 行业深度研究报告
- 复杂技术选型与架构溯源
- 历史/政治/经济事件的多维度推演
- 源码级别的底层运作原理解析

## 多模型适配性与已知副作用

### 对 Gemini 的表现

Gemini 系列**最契合此协议**。原生 Google 搜索整合能完美执行交叉验证，超大上下文窗口能最大化压榨长文本能力。副作用：搜索结果冲突时偶尔输出结构臃肿。

### 对 Claude 的副作用

1. **"过度防御症"**：Claude 可能极度保守，对高置信度常识也打上 `[推测]` 标签。缓解：附加"对高置信度常识无需过度标记"。
2. **信息密度稀释**：为凑篇幅用华丽句式稀释信息。缓解：信息密度检查机制对抗此倾向。
3. **联网缺失**：无搜索能力时按降级路径执行，不影响任务。
